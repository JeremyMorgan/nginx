Bug report #4: autester.py does not handle 301 redirects at all

Priority: Low

Author: Jeff Silverman

Date: 2024-03-03


Symptoms:

When the autotester.py program hits a URL that returns a 301 redirect, the autotester.py ignores the redirect and looks for the string in the redirect message.  Of course, it's not there, so it reports it as a failure.  I think this is a false negative.


Analysis

I am using the httpx package which supports redirection.  Just figure out how it works and implement it.

In the interim, there is a work around.  Have the test_production.tsv detect the string "301".  That will allow it to pass the test.  Then on the next line have the target of the redirect.

Conclusion

This is an easy fix, I think
